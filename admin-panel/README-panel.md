# پنل مدیریت کاربران VPN

این پنل یک وب‌اپلیکیشن کوچک (Flask) روی همین ویندوز است که فایل
`C:\Program Files\OpenVPN\config\users.txt` را مدیریت می‌کند.

## امکانات

- نمایش تمام کاربران با وضعیت:
  - فعال (active)
  - منقضی شده (expired)
  - بدون تاریخ انقضا / تاریخ نامعتبر
- ساخت یا ویرایش کاربر با:
  - یوزرنیم
  - پسورد
  - تعداد روز اعتبار (پیش‌فرض ۳۰ روز)
- غیرفعال کردن کاربر (کامنت‌کردن خط در users.txt)
- تمدید تاریخ انقضای یک کاربر به‌مدت X روز

## پیش‌نیاز

- Python 3 (در سیستم شما نصب است)
- بسته Flask (نصب شده است)

## اجرای پنل (فقط روی همین سرور)

۱. یک پنجره PowerShell معمولی باز کنید و دستور زیر را بزنید:

```powershell
cd "e:\config vpn\admin-panel"
python app.py
```

۲. در مرورگر روی همین سیستم بروید به:

```text
http://127.0.0.1:5000
```

### لاگین پنل

پنل با **Basic Auth** محافظت شده است.

- یوزر پیش‌فرض: `admin`
- پسورد پیش‌فرض: `change-me`

برای امنیت بهتر، قبل از اجرا این دو متغیر محیطی را ست کنید:

```powershell
$env:VPN_ADMIN_USER = "myadmin"
$env:VPN_ADMIN_PASS = "StrongPassword123"
python app.py
```

یا می‌توانید آن‌ها را در System Environment Variables ویندوز به‌صورت دائم
تعریف کنید.

## نکات امنیتی

- **فقط روی localhost** اجرا می‌شود (`127.0.0.1:5000`)؛ از بیرون سیستم
  در دسترس نیست مگر این‌که عمداً آن را reverse-proxy کنید.
- پیشنهاد می‌شود این پنل را فقط روی همان سروری اجرا کنید که OpenVPN نصب شده است.
- پسورد مدیریت (`VPN_ADMIN_PASS`) را حتماً به یک مقدار قوی تغییر دهید.

## فرمت users.txt

پنل از همان فرمتی استفاده می‌کند که اسکریپت `auth.ps1` می‌خواند:

```text
یوزرنیم:پسورد:تاریخ_انقضا
```

- تاریخ انقضا به فرم `YYYY-MM-DD` است (مثال: `2026-04-15`).
- اگر کاربری منقضی شده باشد، در ستون وضعیت «منقضی شده» نمایش داده می‌شود.
- اگر کاربری را از طریق دکمه «غیرفعال» غیر فعال کنید، خط مربوطه در
  `users.txt` با `#` کامنت می‌شود و دیگر قابل استفاده نیست.

## اجرای سرویس به‌صورت پس‌زمینه (اختیاری)

برای این‌که پنل همیشه در حال اجرا باشد، می‌توانید:

- یک Shortcut از دستور اجرای پنل بسازید و در Startup ویندوز قرار دهید، یا
- از NSSM / Task Scheduler برای اجرای خودکار `python app.py` استفاده کنید.

در حالت عادی، اجرای دستی با PowerShell برای مدیریت دوره‌ای کاربران کافی است.
